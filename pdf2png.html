🛡️ Zero data leakage. Your PDF is processed entirely in your browser. No upload, no network requests, no data stored. Content-Security-Policy blocks all outbound connections. Scripts have Subresource Integrity verification.

Powered by pdf.js (Mozilla) and JSZip — both loaded from unpkg CDN with SRI integrity hashes. The page CSP blocks all connect-src, form-action, and frame-src — no data can be exfiltrated.

🔒 How this protects your privacy
  • No uploads. Your PDF is read via the browser File API and never leaves your device. There is no server to upload to — this is a static HTML page on GitHub Pages.
  • No network. The Content-Security-Policy blocks connect-src 'none' — no fetch(), XHR, WebSocket, or navigator.sendBeacon() calls can leave the page. Even if a script tried to exfiltrate data, it couldn't.
  • No forms. form-action 'none' blocks any form submission.
  • No third-party frames. frame-src 'none' prevents embedded content.
  • Script integrity. pdf.js and JSZip are loaded with integrity attributes (SHA-384). The browser checks the file hash matches before executing — a compromised CDN can't inject malicious code.
  • No persistence. No cookies, no localStorage, no IndexedDB are used. All data is held in memory and freed when you click Clear or close the tab.
  • No analytics. No tracking scripts, no analytics, no beacons, no telemetry of any kind.
