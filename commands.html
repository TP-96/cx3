⌨️ CLI Commands hermes [flags] [command]

🚩 Global Flags

FlagDescription
--version, -VShow version
-r, --resume <id>Resume session by ID or title
-c, --continue [name]Resume by name, or most recent session
-w, --worktreeIsolated git worktree mode (parallel agents)
-s, --skills <list>Preload skills (comma-separate or repeat)
-p, --profile <name>Use a named profile
--yoloSkip dangerous command approval
--pass-session-idInclude session ID in system prompt

No subcommand defaults to chat. Pass -q "query" for one-shot mode.

💬 Chat

CommandDescription
hermesInteractive chat (default)
hermes chat -q "query"Single query, non-interactive
hermes chat -m <model>Override model (e.g. anthropic/claude-sonnet-4)
hermes chat -t <toolsets>Comma-separated toolset override
hermes chat --provider <name>Force provider (openrouter, anthropic, nous, etc.)
--verbose, -vVerbose output
--quiet, -QSuppress banner, spinner, tool previews
--checkpointsEnable filesystem checkpoints (/rollback)
--source <tag>Session source tag (default: cli)

💡 Quick Tips

  • First time? Run hermes setup for the interactive wizard
  • Change model mid-stream: /model anthropic/claude-sonnet-4 (or any provider/model)
  • Resume work: hermes --continue picks up your last session
  • One-shot queries: hermes chat -q "Find all TODO comments in my project"
  • Safe mode: hermes --yolo skips command approval prompts
  • Background jobs: /background "Write a unit test for api/auth.py"
  • Context compression: Run /compress manually when the session feels sluggish
  • Session history: hermes sessions list or hermes sessions browse
  • Save skills: After solving a complex problem, say "save this as a skill"
  • Profiles: Use hermes profile create work for isolated configs
  • Voice: /voice on enables voice-to-voice; /voice tts for text-to-speech only
  • Plugins: hermes plugins list to see what's available
